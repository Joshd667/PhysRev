Accessibility Statement

Last updated: 19 November 2025

Our Commitment to Accessibility

The Physics Knowledge Audit Tool is committed to ensuring digital accessibility for all users, including those with disabilities. We strive to continually improve the user experience for everyone and apply relevant accessibility standards.

This statement applies to the Physics Knowledge Audit Tool Progressive Web Application.

Accessibility Standards

We aim to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards. These guidelines explain how to make web content more accessible for people with disabilities and improve usability for all users.

Legal Basis: While we are not a public sector body (and therefore not strictly bound by the Public Sector Bodies Accessibility Regulations 2018), we follow WCAG 2.1 AA as best practice and in line with the Equality Act 2010.

Conformance Status

Current Status: Partially Conformant

"Partially conformant" means that some parts of the content do not fully conform to WCAG 2.1 Level AA. We are actively working to improve accessibility across the application.

Accessibility Features

The Physics Knowledge Audit Tool includes the following accessibility features:

Dark Mode

Automatic light/dark theme switching to reduce eye strain and improve readability

Keyboard Navigation

Most interface elements can be accessed and operated using keyboard only (Tab, Enter, Escape keys)

Responsive Design

Mobile-friendly interface that adapts to different screen sizes and orientations

Colour Contrast

High contrast ratios between text and background colours for improved readability

Visual Focus Indicators

Clear visual indicators showing which element currently has keyboard focus

Semantic HTML

Proper HTML structure with headings, landmarks, and semantic elements for screen readers

Text Resizing

Interface supports browser text resizing up to 200% without loss of functionality

Offline Functionality

Progressive Web App works offline, beneficial for users with unreliable internet connections

Known Accessibility Issues

We are aware of the following accessibility limitations and are working to address them:

Screen Reader Support (Partial)

Some dynamic content updates and interactive components may not be fully announced to screen readers. We are adding ARIA labels and live regions to improve this.

Mindmap Canvas Navigation

The drag-and-drop mindmap canvas is primarily designed for mouse/touch interaction. Keyboard navigation for mindmap editing is limited.

Chart.js Analytics Graphs

Charts in the analytics dashboard may not have sufficient text alternatives for screen reader users. We are investigating data table alternatives.

Mathematical Equation Rendering

KaTeX-rendered equations may not be fully accessible to screen readers. We are exploring MathML alternatives.

Third-Party Content

The application links to external revision resources (videos, notes, simulations). We cannot guarantee the accessibility of third-party content, but we aim to curate resources from providers with good accessibility practices.

Testing and Evaluation

The Physics Knowledge Audit Tool has been tested with:

  • Keyboard-only navigation (Tab, Enter, Escape, Arrow keys)
  • Browser zoom up to 200%
  • Dark mode and light mode colour contrast checks
  • Mobile and desktop screen sizes

Future testing planned: Comprehensive screen reader testing (NVDA, JAWS, VoiceOver), automated accessibility audits (Lighthouse, axe DevTools), and user testing with individuals with disabilities.

Accessibility Feedback

We welcome feedback on the accessibility of the Physics Knowledge Audit Tool. If you encounter accessibility barriers, please let us know:

  • What: Describe the accessibility issue you encountered
  • Where: Which page or feature of the app
  • Device: What device, browser, and assistive technology you were using
  • Contact: Reach out through your educational institution or the app administrator

We aim to respond to accessibility feedback within 5 working days and will work to resolve issues as quickly as possible.

Enforcement and Complaints

If you are not satisfied with our response to your accessibility concerns, you can contact the Equality and Human Rights Commission (EHRC), which is responsible for enforcing the Equality Act 2010.

Technical Specifications

The Physics Knowledge Audit Tool is built using:

  • HTML5, CSS3, JavaScript (ES6+)
  • Alpine.js v3.13.3 for reactive UI
  • TailwindCSS for styling with accessible colour contrast
  • Progressive Web App (PWA) with Service Worker
  • IndexedDB for local data storage

Compatible browsers: Chrome, Firefox, Safari, Edge (latest versions recommended)
